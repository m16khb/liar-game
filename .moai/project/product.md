## 미션과 비전

Liar Game은 온라인 멀티플레이어 게임 플랫폼으로, 사용자들이 실시간으로 라이어 게임을 즐길 수 있는 경험을 제공합니다. 이 프로젝트는 다음의 세 가지 핵심 목표를 추구합니다:

1. **온라인 멀티플레이어 게임 플랫폼**: 실시간으로 즐기는 라이어 게임 웹 플랫폼 구축
2. **소셜 게임 경험 강화**: 친구들과 함께 즐기는 소셜 게임 경험 제공
3. **게임 기술 연구**: WebSocket과 실시간 통신 기술 연구를 통한 기술적 성장

## 주요 사용자

### 1. 일반 게이머
- **특징**: 친구들과 온라인으로 라이어 게임을 즐기려는 일반 사용자
- **핵심 시나리오**:
  - 방 코드를 통해 친구의 방에 참여
  - 실시간으로 다른 플레이어들과 소통하며 게임 진행
  - 모바일 웹에서도 접속하여 게임 즐기기

### 2. 게임 방장
- **특징**: 방을 생성하고 게임을 진행하는 호스트 사용자
- **핵심 시나리오**:
  - 게임 방 생성 및 설정 (제한 시간, 난이도 등)
  - 플레이어 참여 관리 및 게임 시작 통제
  - 비공개 방 운영 및 비밀번호 설정

### 3. 모바일 사용자
- **특징**: 스마트폰이나 태블릿에서 접속하는 사용자
- **핵심 시나리오**:
  - 반응형 웹 인터페이스를 통한 게임 참여
  - 터치 기반의 직관적인 조작
  - 실시간 알림 수신 및 빠른 응답

## 해결할 문제점

### 1. 실시간 동기화 (최우선)
- **문제**: 네트워크 지연으로 인한 게임 상태 불일치
- **해결 방안**: WebSocket 기반의 낮은 지연시간 통신 구현
- **영향도**: 전체 사용자 경험에 직접적인 영향

### 2. 사용자 경험
- **문제**: 복잡한 조작과 직관적이지 않은 인터페이스
- **해결 방안**: 직관적인 UI/UX 설계와 부드러운 게임 플레이 경험 제공
- **영향도**: 사용자 유지率和 만족도

### 3. 확장성
- **문제**: 동시 접속자 증가에 따른 성능 저하
- **해결 방안**: 컨테이너 오케스트레이션을 통한 수평적 확장 지원
- **영향도**: 서비스 안정성과 성장 가능성

### 4. 보안/치팅 방지
- **문제**: 부정 행위를 통한 게임 무결성 훼손 가능성
- **해결 방안**: 서버 측 검증 및 치팅 방지 메커니즘 구현
- **영향도**: 게임의 공정성과 신뢰도

## 차별점

### 1. 기술적 차별점
- **낮은 지연시간**: WebSocket 기반의 최적화된 실시간 통신
- **안정성**: 컨테이너 오케스트레이션을 통한 높은 가용성
- **모던 스택**: TypeScript와 NestJS를 활용한 타입 안전성 확보

### 2. 사용자 경험 차별점
- **직관적 인터페이스**: 최소한의 조작으로 게임 즐기기
- **매끄러운 게임 진행**: 상태 동기화를 통한 끊김 없는 플레이
- **모바일 최적화**: 반응형 디자인을 통한 기기 불문 경험

## 성공 지표

### 기술적 지표
- **지연시간**: P95 < 100ms (WebSocket 메시지)
- **가용성**: 99.9% 업타임 목표
- **동시 접속자**: 1000+ concurrent users 지원

### 비즈니스 지표
- **사용자 참여율**: 일일 활성 사용자 대비 게임 참여율 > 80%
- **세션 지속시간**: 평균 게임 세션 30분 이상
- **모바일 접속률**: 전체 트래픽의 40% 이상 모바일 접속
