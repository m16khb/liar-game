# Opus 고급 작업 실행 커맨드

> inject-context로 로딩된 파일 컨텍스트를 기반으로 Opus 모델의 고급 분석/작업을 실행합니다.

**작업 지시**: $ARGUMENTS

---

## 실행 지침

### Step 1: 컨텍스트 확인

현재 세션에 로딩된 파일 컨텍스트를 확인합니다.
이전 `/inject-context` 커맨드에서 로딩된 파일 내용이 대화 히스토리에 포함되어 있습니다.

### Step 2: 작업 지시 해석

사용자의 작업 지시를 분석합니다:

**작업 지시**: "$ARGUMENTS"

작업 지시가 비어있거나 기본값인 경우:
→ "파일 내용을 분석하고 주요 구조와 핵심 로직을 설명해줘"

### Step 3: 작업 실행

로딩된 컨텍스트를 기반으로 지시된 작업을 수행합니다.

**가능한 작업 유형**:
- 코드 분석 및 설명
- 버그/취약점 탐지
- 리팩토링 제안
- 문서화
- 테스트 코드 생성
- 성능 최적화 제안
- 아키텍처 분석

### Step 4: 결과 보고

작업 결과를 상세하게 보고합니다.

---

## ⚡ 실행 지시

**지금 바로 아래를 실행하세요:**

1. 대화 히스토리에서 이전에 로딩된 파일 컨텍스트 확인
2. "$ARGUMENTS" 작업 지시 해석
3. 해당 작업을 철저하게 수행
4. 결과를 명확하고 구조화된 형태로 보고

**Opus 모델의 강점을 활용하세요:**
- 복잡한 코드 패턴 인식
- 깊이 있는 분석과 추론
- 고품질 코드 생성
- 포괄적인 문제 해결
